Таблица Experiment

id (PK)	- Уникальный идентификатор эксперимента в базе данных.
name - Название эксперимента, например "SpamClassifier" или "ImageSegmentation".
created_at - Дата и время создания эксперимента.
description - Краткое описание эксперимента, например цель или особые примечания.

 Таблица ExperimentRun

id (PK)	- Уникальный идентификатор конкретного запуска/версии эксперимента.
experiment_id (FK)	- Внешний ключ, указывающий на эксперимент из таблицы Experiment.
version	Номер версии запуска. Например 1, 2, 3 для разных запусков одного эксперимента.
started_at	- Время старта конкретного запуска.
status	- Статус запуска: "running", "finished", "failed" или другие состояния.

Таблица Parameters

id (PK)	- Уникальный идентификатор параметра.
run_id (FK)	- Внешний ключ, указывающий на ExperimentRun.
key	- Название параметра, например "learning_rate", "batch_size".
value	- Значение параметра для конкретного запуска, например 0.001 или 32.

Таблица Metrics

id (PK)	- Уникальный идентификатор метрики.
run_id (FK)	- Внешний ключ, указывающий на ExperimentRun.
key	- Название метрики, например "accuracy", "loss".
value	- Значение метрики для конкретного запуска, например 0.87.
timestamp	- Время, когда метрика была зафиксирована. Позволяет отслеживать динамику во времени.

Таблица Artifacts

id (PK)	- Уникальный идентификатор артефакта.
run_id (FK)	- Внешний ключ, указывающий на ExperimentRun.
file_path	- Путь к файлу артефакта (модель, лог, график).
type	- Тип артефакта: "model" (сохранённая модель), "plot" (график), "log" (лог-файл) и т.д.



Experiment = сам проект/идея
ExperimentRun = конкретный запуск или версия эксперимента
Parameters, Metrics, Artifacts = всё, что относится к конкретной версии (связь через run_id)
